# 🧠 Prompt Engineering Tool v1.0

Професійний, статичний **інструмент для структурування промптів** та **валідації відповідей ШІ** через QA-чекліст, підрахунок балів і рекомендації щодо покращення.

> Дизайн виконано у стилі **Modern Soft Tech**: світлий фон, індиго/пурпурні акценти, м’які картки та шрифт Inter.

## Основні можливості

- **Генератор промпту**: збирає структурований промпт із форми (роль, тип задачі, контекст, обмеження, формат).
- **Валідація контексту**: лічильник символів + блокування кнопки генерації, поки контекст < 20 символів.
- **QA checklist**: 8 критеріїв якості відповіді + **скоринг**.
- **Progress bar**: візуально показує відсоток проходження QA.
- **Розумні поради**: 2–4 рекомендації, сформовані правилами на основі невідмічених пунктів.
- **Copy/Clear для промпту**: швидке копіювання в буфер та очищення поля (кнопки поверх textarea).

## Логіка проєкту (як усе пов’язано)

### 1) Ввід → Валідація → Кнопка генерації

- Поле `#context` слухає подію `input`.
- Функція **`updateCharCount()`** оновлює бейдж `#charCount` у форматі `N / 20` і підсвічує стан:
  - `< 20` → `text-danger`
  - `>= 20` → `text-success`
- Функція **`updateGenerateButtonState()`** керує `#generateBtn`:
  - `disabled = true`, якщо символів < 20
  - `disabled = false`, якщо символів >= 20

### 2) Генерація промпту

Кнопка **Generate prompt** викликає **`generatePrompt()`**, яка будує текст із полів форми у фіксованому форматі:

```text
Role: <значення з select>
Task: <значення з select>
Context:
<контекст>

Constraints:
<обмеження або (none)>

Output format: <значення з select>
```

Важливо: значення `<option>` у select **не перекладаються**, оскільки вони використовуються як стабільні значення для генерації.

### 3) QA checklist → Score + Badge + Progress bar

- Кожен чекбокс має клас `.qa-item`.
- На `change` викликається **`calculateScore()`**:
  - **вага чекбоксів однакова**: 1 пункт = 1 “passed”
  - формується `passed/total` (наприклад, `6/8 passed`)
  - обчислюється відсоток \(pct\)
- UI-індикатори:
  - `#scoreText` → `passed/total passed`
  - `#scoreBadge` → `pct%` + колір (Bootstrap badge):
    - `>= 75%` → success
    - `50–74%` → warning
    - `< 50%` → danger
  - `#scoreProgress` → ширина progress bar (`width: pct%`)

### 4) Suggest improvements (масив правил)

Кнопка **Suggest improvements** викликає **`suggestImprovements()`**:

- Збирає список `unchecked` за `id` чекбоксів.
- Для кожного невідміченого критерію додає “правильну” рекомендацію (до 4 шт.).
- Якщо все відмічено — показує корисні “next-level” поради.

Це **локальна логіка без API**: рекомендації — це заздалегідь прописані правила, що дає прогнозований результат і швидку роботу.

### 5) Copy to clipboard

Функція **`copyToClipboard()`**:
- спочатку пробує `navigator.clipboard.writeText`
- якщо недоступно — використовує fallback через тимчасовий `<textarea>` + `document.execCommand('copy')`

## Інструкція запуску

Проєкт статичний — сервер не потрібен.

1. Відкрийте файл `index.html` у браузері (подвійний клік або Open With…).
2. Заповніть **Контекст** (мін. 20 символів) і натисніть **Generate prompt**.
3. Скопіюйте промпт, вставте відповідь ШІ у “Відповідь AI”, пройдіть чекліст та отримайте підказки.
